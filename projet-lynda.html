Lynda — Assistante IA

Lynda est l'assistante conversationnelle intégrée à ce portfolio. Elle répond en langage naturel, génère des images, recherche sur le web et guide les visiteurs dans l'univers de Maurel Brou — le tout propulsé par une clé API Groq, côté serveur.

Objectifs du projet

  • Offrir un accompagnement conversationnel en temps réel aux visiteurs.
  • Rester strictement focalisée sur l'univers « Maurel Brou ».
  • Générer des images et synthétiser des recherches web.
  • Garder la clé API hors de portée du navigateur (sécurité).

Fonctionnalités clés

  • Réponses IA générées via Groq (modèle llama-3.3-70b-versatile).
  • Génération d'images par IA directement dans le chat.
  • Recherche web (DuckDuckGo) pour les infos récentes.
  • Affichage progressif (effet de frappe) et mémoire de conversation.

Architecture

Le système repose sur une séparation nette entre le frontend (navigateur) et un backend Python léger. La clé API Groq ne quitte jamais le serveur : le navigateur parle uniquement à un proxy local, qui relaie la requête vers Groq.

  • Frontend — assets/js/agent.js : interface de chat, détection d'intention, affichage progressif et rendu des images générées.
  • Backend — backend/server.py (Flask) : expose /api/chat (Groq), /api/image (génération d'images) et /api/search (recherche web).
  • Clé API Groq — stockée côté serveur dans GROQ_KEY, jamais exposée au client.
  • Modèle de secours — llama-3.1-8b-instant en cas de rate-limit du modèle principal.

Flux typique : Utilisateur → agent.js → backend Flask → Groq API → réponse → affichage progressif.

La clé API Groq

Lynda utilise l'API Groq (tier gratuit généreux, ~30 requêtes/min) comme fournisseur d'IA principal. La clé est configurée dans le backend :

  • Déclarée via la variable d'environnement GROQ_KEY ou directement dans server.py.
  • Jamais envoyée au navigateur : le proxy backend ajoute l'en-tête Authorization: Bearer … côté serveur.
  • Modèle par défaut : llama-3.3-70b-versatile (rapide, de qualité, gratuit).

Pour lancer le backend : python backend/server.py (écoute sur http://127.0.0.1:5000).
